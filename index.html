Othm Symbols

A header only library for keeping things simple

What are symbols (and why do I want to use them?)

Symbols are (often) pointers (in this case of type struct othm_symbol_struct *) that have symbolic meaning making them useful for constants. In this case they also inherit from struct othm_request making them ideal for being used as hashmap keys.

Installation

 make install

How to use

Include the header:

#include <othm_symbols.h>

Add a new symbol (outside of a function, since symbols should be global):

#ifndef OTHM_SYMBOL_SYMBOL_my_symbol
#define OTHM_SYMBOL_SYMBOL_my_symbol
OTHM_SYMBOL_INIT(my_symbol);
#endif

Or if you prefer a keyword symbol:

#ifndef OTHM_SYMBOL_KEYWORD_my_keyword
#define OTHM_SYMBOL_KEYWORD_my_keyword
OTHM_KEYWORD_INIT(my_keyword);
#endif

You can compare (keyword/symbol/any symbol type) symbols:

OTHM_SYMBOL(my_symbol) == OTHM_SYMBOL(my_symbol);  /* ==> 1 */
OTHM_KEYWORD(my_symbol) == OTHM_SYMBOL(my_symbol); /* ==> 0 */

You can get their names (char *):

OTHM_KEYWORD_NAME(my_keyword)

You can print them out in a lisp inspired fashion:

othm_symbol_print(OTHM_SYMBOL(my_symbol));   /* ==> 'my_symbol  */
othm_symbol_print(OTHM_KEYWORD(my_keyword)); /* ==> :my_keyword */

Symbols can added to a global hashmap making them accessible at run time:

int main(void)
{
othm_symbols_init_runtime();
OTHM_SYMBOL_ALLOW_AT_RUNTIME(hi);
OTHM_KEYWORD_ALLOW_AT_RUNTIME(bye);
if (othm_symbol_get_from_string("hi") == OTHM_SYMBOL(hi))
        othm_symbol_print(othm_keyword_get_from_string("bye"));
}
